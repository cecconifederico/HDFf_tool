# Results Structure

This directory contains the outputs and data structures generated by the HFDf estimation pipeline across the neurophysiological datasets (`A1`, `M1`, and `S1`).

The execution script aggregates of the multi-channel processing into a MATLAB table named `HFDf_totale_1`, which is saved inside the workspace file (`HFDf_totale_curves.mat` or `HFDf1.mat`).

## Dataset Dimensions
The output matrix populates exactly **47 rows**, corresponding to the total number of recording channels across the target cortical regions:
- **A1 (Auditory Cortex):** 10 channels (Rows 1 to 10)
- **M1 (Primary Motor Cortex):** 20 channels (Rows 11 to 30)
- **S1 (Primary Somatosensory Cortex):** 17 channels (Rows 31 to 47)

---

## Output Data Table Schema (`HFDf_totale_1`)

The table consists of **7 distinct informational columns** :

| Column Name | Data Type | Analytical Description / Meaning | Example |
| :--- | :--- | :--- | :--- |
| **Area** | `string` / `cell` | The specific anatomical cortical area of the recording. | `'A1'`, `'M1'`, `'S1'` |
| **Canale** | `string` / `cell` | Alphanumeric unique global identifier of the target channel. | `'GD013Lu_8W'` |
| **TipoElettrodo** | `string` / `cell` | Technical classification extracted from the first 2 letters of the channel name. | `'GD'` |
| **SoggettoID** | `string` / `cell` | Anonymized Subject ID extracted from the 3 central numeric digits. | `'013'` |
| **Elettrodo** | `string` / `cell` | Local physical identification of the electrode (remaining central string before the state). | `'Lu_8'` |
| **HFD_vs_K** | `double` (vector) | Vector array storing the discrete Higuchi Fractal Dimension ($HFD_f$) evaluated for each channel while scaling $k$ from $k_{min}$ to $k_{max}$. | `[2, 3, 4, ...,35]` |
| **k_range** | `double` (vector) | The independent scale factor step vector used for the iteration. | `[2, 3, 4, ..., 35]` |

---

## Automated Channel Label Parsing Rule

Taking **`GD013Lu_8W`** as a standard gold-reference example, the structural partitioning behaves as follows:

1. **`GD` (Characters 1–2) ➔ `TipoElettrodo`:** Identifies the type of electrode.
2. **`013` (Characters 3–5) ➔ `SoggettoID`:** Represents the unique three-digit numerical identifier code for the experimental subject.
3. **`Lu_8` (Remaining center sub-string) ➔ `Elettrodo`:** Defines the physical location or specific spatial coordinate label of that sensor within the brain cluster.
4. **`W` (Final Character) ➔ `Stato`:** Denotes the current monitoring phase or experimental/pathological state of the recording block.

---

## Methodological Notes on `HFD_vs_K` Calculation
1. **Segment Median Windowing:** For each individual channel, the raw continuous signal (13,600 data points per session) is processed dynamically. To extract a stable trajectory that defines the structural complexity profile of the channel across the scale spectrum ($k \in [k_{min} \dots k_{max}]$), a median is applied over the calculated sub-segments.
2. **Statistical Quality Control:** If a specific channel displays a poor linear regression fit ($R^2 < 0.90$) due to experimental artifacts or noise, its `HFD_vs_K` array entries are safely forced to `NaN` placeholders to protect the downstream statistical comparison from anomalous outliers.
